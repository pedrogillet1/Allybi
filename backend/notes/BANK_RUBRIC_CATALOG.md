## Grading Dimensions Explained

| Dimension | Definition | Scoring |
|-----------|-----------|---------|
| **SSOT Integrity** | Bank is registered in bank_registry, has checksum, has alias, no duplicates | 0 = missing from registry; 1 = fully registered with checksum |
| **Coverage** | All expected entries/rules/patterns are present, no empty arrays | 0-1 scale: count of actual / count of expected |
| **Precision** | No overly broad patterns, no collision clusters, no placeholder content | 0-1 scale: 1 - (collision_clusters / total_patterns) |
| **Locale Parity** | EN and PT banks have matching counts and key coverage | 0-1 scale: min(EN,PT) / max(EN,PT) |
| **Runtime Wiring** | Bank is loaded by a production service via getBank() or equivalent | 0 = DEAD; 0.5 = test-only; 1 = WIRED |
| **Deploy Readiness** | Checksum present, registry entry correct, no CI gate failures | 0-1 scale: pass/fail of checksum + registry + schema gates |
| **Collision Risk** | Intent patterns: risk of multiple patterns matching same input | 0 = high risk (>5 clusters); 0.5 = medium (2-5); 1 = low (<2) |
