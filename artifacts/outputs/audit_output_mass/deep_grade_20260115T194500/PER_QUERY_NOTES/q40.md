## Root Cause Analysis

### Primary: lastDocumentIds Not Used in Retrieval

The query is a direct follow-up to q18 about the Integration Guide. The retrieval should have boosted chunks from that document.

**Evidence:**
- q18 returned `Koda_Integration_Guide_5_Presentation_&_Document_Ge...` as source
- q40 returned `Capítulo 8 (Framework Scrum).pdf` - completely different document
- Query says "o guia" (the guide) - clear reference to q18's Integration Guide

### Secondary: Semantic Search Mismatch

The embedding similarity search found Scrum chapter more similar than Integration Guide due to generic terms.

**Evidence:**
- "separar conteúdo de layout" matched Scrum's workflow separation concepts
- Integration Guide content about "content/layout separation" not boosted

### Tertiary: Follow-up Reference Missed

The phrase "o guia" should have triggered follow-up detection linking to q18.

**Evidence:**
- Conversation plan marks `dependsOn: "q18"`
- q18 discussed "Integration Guide 5"
- q40's "o guia" is a clear anaphoric reference

## Minimal Fix

**File:** `backend/src/services/core/kodaRetrievalEngineV3.service.ts`

In `retrieveWithMetadata()` function:

```typescript
async retrieveWithMetadata(query: string, context: ConversationContext) {
  // Get raw retrieval results
  let chunks = await this.hybridSearch.search(query, context.userId);

  // CRITICAL FIX: Boost chunks from lastDocumentIds
  if (context.lastDocumentIds?.length > 0) {
    chunks = chunks.map(chunk => {
      if (context.lastDocumentIds.includes(chunk.documentId)) {
        return {
          ...chunk,
          score: chunk.score * 1.5, // 50% boost for context documents
          boosted: true
        };
      }
      return chunk;
    });

    // Re-sort by boosted scores
    chunks.sort((a, b) => b.score - a.score);
  }

  return chunks.slice(0, this.maxChunks);
}
```
