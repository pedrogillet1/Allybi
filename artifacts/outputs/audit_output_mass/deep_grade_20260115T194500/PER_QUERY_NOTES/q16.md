## Root Cause Analysis

### Primary: Follow-up Phrase Detection Failure

The query contains "nesses números" (these numbers) - a clear follow-up reference to q12's EBITDA numbers. This should have triggered follow-up intent inheritance.

**Evidence:**
- Follow-up phrase "nesses números" present
- Query marked as `dependsOn: "q12"` in conversation plan
- Intent engine returned `extraction` instead of `documents`

### Secondary: Extraction Intent Bypass

Same issue as q14 - `extraction` intent bypasses the `wasDocContext` check in decision tree.

**Evidence:**
- `intent: "extraction"` in results.jsonl
- Previous query (q12) had `intent: "documents"`
- Latency 12ms indicates instant template (no RAG)

1. **Follow-up Detection** (`routingPriority.service.ts`)

## Minimal Fix

**File:** `backend/src/services/core/decisionTree.service.ts`
**Location:** Line 294

```typescript
// BEFORE
case 'extraction':
  return 'extraction';

// AFTER
case 'extraction':
  if (wasDocContext && hasDocs && !hasExplicitHelpKeyword) {
    return 'documents';
  }
  return 'extraction';
```
